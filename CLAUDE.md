# 법령 모니터링 작업 규칙

이 저장소는 법률·위생·세무처럼 정확성이 중요한 자료다.

1. 새 사실은 국가법령정보센터, 정부부처, 공공기관 등 1차 자료를 우선한다.
2. 기사만 확인된 내용은 반드시 🟡 검토로 표시하고 확정 표현을 쓰지 않는다.
3. 마스터 문서를 수정하기 전에 같은 주제의 모든 문구를 검색한다. 본문·체크리스트·팩트표·변경이력의 상태가 서로 다르면 먼저 모순을 해소한다.
4. 각 주장에 출처 URL과 확인일을 남긴다. 링크가 열리지 않거나 본문을 확인하지 못하면 ✅로 올리지 않는다.
5. `python3 scripts/check_fact_consistency.py`를 실행해 통과한 뒤 커밋한다.
6. 실제 변경이 없으면 날짜만 바꾸는 커밋은 만들지 않는다.
7. GitHub Issue의 `[FABLE]`, `[CODEX]` 담당 표시를 확인하고, 작업 종료 시 Issue/PR에 변경 내용·검증 결과·남은 의문을 기록한다.
